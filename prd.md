## 1. 产品概述与目标

### 1.1 是什么

OpenClaw Wallpaper 是一个 Windows 桌面动态壁纸应用，将 OpenClaw 的 Agent 管理/监控/交互界面以**星露谷物语风格的像素风 2D 场景**呈现在桌面上。用户可以在桌面壁纸中直观看到 Agent 工作状态、与 Agent 对话、管理 OpenClaw 服务。

### 1.2 为什么做

- **降低 OpenClaw 使用门槛** — 将命令行工具变成可视化、拟人化的桌面伴侣
- **提升用户粘性** — 7×24 小时在桌面可见，持续强化产品存在感
- **建立情感连接** — 像素风小人赋予 Agent 人格化的存在感，让"管理 AI"变成"和小伙伴协作"
- **为后续商业化铺路** — 皮肤、场景、主题包等虚拟商品天然适合壁纸场景（MVP 不涉及）

### 1.3 MVP 目标

> **一句话：一个能管理 OpenClaw 的星露谷风格动态壁纸。**

具体目标：
1. ✅ 壁纸在桌面正常运行，不干扰日常使用
2. ✅ 实时展示 OpenClaw Agent/Session 的运行状态
3. ✅ 支持基础交互：查看 Agent 信息、与 Agent 对话
4. ✅ 支持 OpenClaw 服务的基础管控（检测、启动、停止）
5. ✅ 视觉风格统一：星露谷物语式的像素风温馨场景

### 1.4 MVP 不做

| 功能 | 原因 |
|------|------|
| 支付 / 充值 | 后期迭代 |
| 用户登录 / 账户系统 | 后期迭代 |
| 商店系统 | 后期迭代 |
| 换装 / 皮肤系统 | 后期迭代 |
| macOS / Linux 支持 | 后期迭代，MVP 只做 Windows |
| 多场景切换 | 后期迭代，MVP 只做一个默认场景 |
| 日夜交替 / 天气系统 | 后期迭代，MVP 只做固定白天场景 |
| 音效 / 背景音乐 | 后期迭代 |
| OpenClaw 自动安装 / 捆绑安装 | 后期迭代，MVP 假设用户已安装 OpenClaw |

## 2. 用户故事 / 使用场景

### US-1：看见 Agent 在工作

> 作为 OpenClaw 用户，我希望在桌面上直观看到我的 Agent 正在工作，这样我不用打开终端就知道任务进展。

**验收标准：**
- 当 OpenClaw 有活跃 session 时，对应的像素角色在场景中播放"工作"动画
- 当 session 完成时，角色回到"空闲"状态
- 当 session 出错时，角色显示"错误"状态

### US-2：查看 Agent 详情

> 作为 OpenClaw 用户，我希望点击桌面上的某个 Agent 小人，能看到它的基本信息和当前任务状态。

**验收标准：**
- 鼠标点击角色后弹出信息面板
- 面板展示：Agent 名称、当前状态（空闲/工作中/错误）、正在处理的任务摘要
- 面板可关闭

### US-3：和 Agent 对话

> 作为 OpenClaw 用户，我希望能在桌面上直接和 Agent 对话，而不是切到终端。

**验收标准：**
- 从 Agent 信息面板可打开对话窗口
- 对话窗口支持文字输入、发送、接收回复
- 对话通过 OpenClaw Gateway API 实际执行

### US-4：管理 OpenClaw 服务

> 作为 OpenClaw 用户，我希望在壁纸中能看到 OpenClaw 是否在运行，并能启动/停止它。

**验收标准：**
- 场景中有明确的视觉指示：OpenClaw 在线（亮灯/烟囱冒烟） vs 离线（灭灯/无烟）
- 系统托盘菜单支持启动/停止 OpenClaw Gateway
- OpenClaw 离线时，Agent 角色全部进入"休眠"状态

### US-5：不影响日常使用

> 作为用户，我希望这个壁纸不会拖慢我的电脑，前台工作时它应该安静地待在后面。

**验收标准：**
- 前台有其他窗口时自动降帧至 ≤5 FPS 或暂停渲染
- CPU 占用空闲时 <2%，GPU 占用空闲时 <5%
- 内存占用 <200MB

### US-6：独立窗口模式

> 作为用户，我有时候不想用壁纸模式，想把场景当作一个普通窗口打开。

**验收标准：**
- 系统托盘菜单可切换：壁纸模式 ↔ 独立窗口模式
- 独立窗口模式下交互不受壁纸层限制（完整鼠标键盘支持）
- 两种模式共享场景状态和数据

## 4. 场景设计说明

### 4.1 风格定义

**关键词：** 星露谷物语、像素风、温馨、田园、工作坊

- **像素密度：** 32×32 px 为基础 tile 尺寸，角色 32×48 px
- **调色板：** 温暖的大地色系为主，明亮但不刺眼，参考星露谷的春季/夏季色调
- **视角：** 俯视 45°（等距视角的简化版），类似 RPG Maker / 星露谷物语的 ¾ 鸟瞰视角
- **氛围：** 阳光明媚的午后，一个温馨的小型 AI 工作坊

### 4.2 场景布局

场景是一个横向铺满桌面的连续画面，从左到右分为几个功能区域：

```
┌────────────────────────────────────────────────────────────┐
│  ☁️         ☁️                    ☁️          🌤️           │  天空层
│                                                            │
│  🌳    ┌──────────┐      ┌──────────┐       🌻  🌳       │  建筑 & 植被
│       │ 🏠 工作坊  │      │ 📬 收发站  │                    │
│       │ (Working)  │      │ (Comms)   │    ┌─────┐         │
│       │            │      │           │    │🏕️休息│         │  功能建筑
│       └──────────┘      └──────────┘    │ 区    │         │
│                                          └─────┘          │
│  🧑‍💻  🧑‍💻   ── 小路 ──── 🧑‍💻 ── 小路 ────  🧑‍💻  🧑‍💻        │  角色层
│                                                            │
│  🌿 🌿  🪨   💐  🌿  🌾  🌾   💐  🌿  🪨  🌿  🌿       │  地面装饰
│~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~│  水面/边界
└────────────────────────────────────────────────────────────┘
```

### 4.3 场景元素详解

| 区域 | 名称 | 描述 | 功能对应 |
|------|------|------|---------|
| 🏠 | **工作坊** | 像素风木质小屋，有窗户和烟囱 | Agent 工作区域。正在工作的 Agent 在此建筑前/内播放工作动画 |
| 📬 | **收发站** | 带邮箱的小亭子 | 消息/对话区域。有新消息时信箱有提示。点击可查看消息 |
| 🏕️ | **休息区** | 篝火旁有木椅和帐篷 | 空闲 Agent 在此区域闲逛或坐着休息 |
| 🌳🌻 | **植被** | 树木、花朵、灌木 | 纯装饰，增加田园氛围 |
| 小路 | **石板路** | 连接各建筑的小路 | Agent 移动路径（P2 行走动画使用） |
| 🪨🌿 | **地面装饰** | 石头、草丛、蘑菇 | 纯装饰 |
| ☁️🌤️ | **天空** | 蓝天白云 | 纯背景，MVP 固定不变 |

### 4.4 OpenClaw 状态视觉指示

| OpenClaw 状态 | 场景表现 |
|---------------|---------|
| **在线运行中** | 工作坊烟囱冒烟 ✅，窗户亮灯 ✅，场景色调明亮 |
| **离线** | 烟囱无烟 ❌，窗户灭灯 ❌，场景色调略暗（加一层半透明灰色遮罩），所有角色进入"休眠"姿势 |

### 4.5 场景渲染层级（从下到上）

1. **天空背景层** — 静态或缓慢滚动的云朵
2. **远景层** — 远处山丘/树木（可做微弱视差）
3. **建筑层** — 工作坊、收发站、休息区
4. **地面层** — 小路、草地、装饰物
5. **角色层** — Agent 像素小人
6. **特效层** — 粒子效果（烟囱烟雾、状态光效）
7. **UI 覆盖层** — 信息面板、对话气泡、状态指示

## 5. Agent 角色设计

### 5.1 角色基础规格

- **尺寸：** 32×48 px（2 tile 宽 × 3 tile 高），适合像素风比例
- **风格：** 星露谷式 Q 版小人，大头身比例（约 1:2 头身比）
- **颜色区分：** 每个 Agent 有不同的主色调（通过衣服/帽子颜色区分）
- **MVP 角色数量：** 1 套基础角色模板，通过调色板交换生成不同外观

### 5.2 动画状态机

```
                    ┌──────────┐
         ┌─────────│  Spawn   │──────────┐
         │         │ (出现)    │          │
         │         └──────────┘          │
         ▼                               ▼
  ┌────────────┐                  ┌────────────┐
  │   Idle     │◄────完成────────│  Working   │
  │  (空闲)    │                  │  (工作中)   │
  │            │─────开始任务────►│            │
  └─────┬──────┘                  └─────┬──────┘
        │                               │
        │                               │ 出错
        │                               ▼
        │                        ┌────────────┐
        │                        │   Error    │
        │◄────恢复/手动重置──────│  (错误)    │
        │                        └────────────┘
        │
        │ session 结束
        ▼
  ┌────────────┐
  │  Despawn   │
  │  (消失)    │
  └────────────┘
```

### 5.3 各状态动画描述

| 状态 | 动画帧数 | 动画描述 | 循环 |
|------|---------|---------|------|
| **Spawn（出现）** | 6-8 帧 | 从地面弹出/从光点中凝聚出现，带小星星粒子特效 | 一次性 |
| **Idle（空闲）** | 4-6 帧 | 站立微微晃动，偶尔眨眼、转头、挠头，呈放松姿态 | 循环，带随机变体 |
| **Working（工作中）** | 6-8 帧 | 角色面朝工作台/电脑，双手快速敲击，头顶有齿轮或代码符号冒出 | 循环 |
| **Error（错误）** | 4-6 帧 | 角色头顶冒出 `!` 或 `?` 符号，身体僵硬微颤，表情变为 `>_<` | 循环 |
| **Despawn（消失）** | 6-8 帧 | 向上飘散成像素粒子 / 走出画面 | 一次性 |

### 5.4 MVP 美术实现方案

**Spritesheet 帧动画**（非 Spine 骨骼动画，降低 MVP 美术成本）：

- 每个状态制作一组 PNG Spritesheet
- 使用 TexturePacker 或类似工具打包
- PixiJS `AnimatedSprite` 播放
- 角色差异化通过**调色板交换（Palette Swap）** 实现：
  - 1 套基础灰度 Spritesheet
  - N 个调色板 JSON（蓝色 Agent、绿色 Agent、红色 Agent…）
  - 运行时通过 PixiJS Color Replace Filter 或 shader 换色

这样 MVP 阶段只需绘制 **1 套完整角色动画**，即可生成多个外观不同的 Agent。

### 5.5 角色名称显示

- 每个角色头顶显示其 Agent/Session 名称（像素字体，如 "PM"、"Frontend"、"Analyst"）
- 字体使用像素风位图字体（如 Press Start 2P 或自制像素字体）
- 名称背景半透明黑色，确保在各种背景上可读

## 6. 交互设计

### 6.1 双模式架构

| 模式 | 描述 | 交互能力 | 切换方式 |
|------|------|---------|---------|
| **壁纸模式** | 场景渲染在桌面 WorkerW 层，位于图标下方 | 桌面聚焦时支持鼠标点击；前台有窗口时无法直接交互 | 系统托盘 → "切换为窗口模式" |
| **独立窗口模式** | 场景作为普通 Tauri 窗口运行（可调大小、最小化） | 完整鼠标键盘支持，不受限制 | 系统托盘 → "切换为壁纸模式" |

### 6.2 壁纸模式交互流程

```
用户看到桌面壁纸上的 Agent 小人
    │
    ├─ 无前台窗口（桌面已聚焦）
    │   ├─ 鼠标悬停角色 → 角色高亮 + 名称/状态 tooltip
    │   ├─ 左键单击角色 → 弹出 Agent 信息面板（PixiJS UI 覆盖层）
    │   │   ├─ 面板显示：名称、状态、任务摘要
    │   │   └─ 面板按钮："对话" → 打开独立对话窗口（Tauri 新窗口，正常 Z-order）
    │   └─ 右键单击角色 → 快捷菜单（查看详情 / 对话）
    │
    └─ 前台有窗口覆盖
        └─ 壁纸层不可交互 → 用户可：
            ├─ 通过全局热键（如 Ctrl+Shift+W）最小化所有窗口，聚焦桌面
            └─ 通过系统托盘切换到独立窗口模式
```

### 6.3 对话窗口

对话通过**独立的 Tauri 窗口**实现（不在壁纸层内），原因：
- 壁纸层键盘输入支持受限
- 独立窗口有完整的文字输入体验
- 避免壁纸层 z-order 问题

**对话窗口设计：**
- 像素风 UI 风格（与壁纸场景视觉统一）
- 顶部：Agent 名称 + 状态
- 中部：对话历史（消息气泡，支持滚动）
- 底部：文字输入框 + 发送按钮
- 窗口大小：约 400×600 px，可调节
- 通过 OpenClaw Gateway HTTP API 发送消息，WebSocket 接收回复流

### 6.4 系统托盘菜单

```
OpenClaw Wallpaper
├─ 🟢 OpenClaw: Running          ← 状态指示
├─ ─────────────────
├─ 切换为窗口模式 / 壁纸模式     ← 模式切换
├─ 启动 OpenClaw / 停止 OpenClaw ← 服务控制
├─ ─────────────────
├─ 设置                           ← 打开设置窗口
└─ 退出                           ← 还原壁纸并退出
```

### 6.5 壁纸还原

退出应用时，必须：
1. 调用 `tauri-plugin-wallpaper` 的 detach 方法
2. 还原用户原有桌面壁纸
3. 确保不残留进程

## 7. OpenClaw 集成需求

### 7.1 连接方式

| 层级 | 协议 | 用途 |
|------|------|------|
| 前端 → OpenClaw Gateway | HTTP REST | 查询 sessions 列表、session 详情、发送对话消息 |
| 前端 → OpenClaw Gateway | WebSocket | 实时订阅 session 状态变化、新消息推送 |
| Rust Backend → 本地进程 | 进程管理 | 检测/启动/停止 OpenClaw Gateway 进程 |

### 7.2 Gateway 检测

**启动流程：**
```
应用启动
  ├─ Rust 侧：检查 OpenClaw Gateway 是否在运行
  │   ├─ 方式 1: HTTP 请求 http://localhost:{port}/api/health（或类似健康检查端点）
  │   └─ 方式 2: 检查进程列表中是否有 openclaw gateway 进程
  │
  ├─ Gateway 在线 → 前端建立 WebSocket 连接，拉取初始 sessions 数据
  └─ Gateway 离线 → 场景进入"离线模式"（灰暗色调），提示用户启动
```

**持续检测：**
- WebSocket 断线后自动重连（指数退避：1s → 2s → 4s → 8s → 30s max）
- 重连成功后全量刷新 sessions 状态
- 每 30s 一次 HTTP 心跳作为 WebSocket 的备用检测

### 7.3 Sessions/Agent 数据映射

| OpenClaw 概念 | 壁纸中的表现 |
|---------------|-------------|
| 活跃 Session | 场景中的一个角色小人 |
| Session label | 角色头顶名称文字 |
| Session 状态 (active/idle) | 角色动画状态（Working / Idle） |
| Session 错误 | 角色 Error 动画 |
| 新 Session 创建 | 新角色 Spawn 动画出现 |
| Session 结束 | 角色 Despawn 动画消失 |
| Session 消息 | 角色头顶气泡（P2） |

### 7.4 OpenClaw 启停控制

通过 Tauri Rust 侧管理：
- **启动：** `spawn` OpenClaw Gateway 进程（`openclaw gateway start` 或等效命令）
- **停止：** 向 Gateway 发送停止信号（`openclaw gateway stop` 或等效命令）
- **重启：** 停止 + 启动

注意事项：
- 不修改 OpenClaw 配置文件（AGENTS.md 规则：不碰系统配置）
- 不管理 OpenClaw 的安装/更新（MVP 假设已安装）
- 只管理 Gateway 进程的生命周期

### 7.5 对话集成

```
用户输入文字
  ↓
前端发送 HTTP POST 到 OpenClaw Gateway
  POST /api/sessions/:key/message
  body: { content: "用户的消息" }
  ↓
Gateway 处理并返回
  ↓
WebSocket 推送 Agent 回复
  ↓
前端对话窗口显示回复内容
```

MVP 对话功能范围：
- ✅ 发送文字消息
- ✅ 接收文字回复（流式显示）
- ✅ 查看历史消息
- ❌ 不支持发送图片/文件
- ❌ 不支持语音

## 8. 性能指标

### 8.1 目标指标

| 指标 | 壁纸模式（空闲） | 壁纸模式（有交互） | 独立窗口模式 |
|------|-----------------|-------------------|-------------|
| **CPU** | <2% | <5% | <8% |
| **GPU** | <5% | <10% | <15% |
| **内存** | <150MB | <200MB | <250MB |
| **帧率** | 10-15 FPS | 30 FPS | 30 FPS |
| **启动时间** | <3s | - | <3s |
| **包体大小** | <30MB（安装包） | - | - |

### 8.2 性能策略

| 策略 | 触发条件 | 行为 |
|------|---------|------|
| **前台暂停** | 检测到前台窗口全屏覆盖桌面 | 帧率降至 1 FPS 或暂停渲染 |
| **空闲降帧** | 无用户交互 >30s | 帧率降至 10-15 FPS |
| **交互唤醒** | 用户鼠标移入壁纸区域 | 帧率升至 30 FPS |
| **渲染跳帧** | 场景无任何变化（无动画帧更新） | 跳过本帧渲染，复用上一帧 |
| **资源按需加载** | 角色数量变化 | 动态加载/卸载 Spritesheet 资源 |

### 8.3 内存管理

- 角色 Spritesheet 使用 PixiJS Texture Cache，不重复加载
- 场景 Tile 贴图打包为 Atlas，减少 draw call
- 对话历史只缓存最近 50 条，更早的消息按需从 API 加载
- 定期检查内存使用，超过阈值时清理非可见资源缓存

## 9. 技术约束与限制

### 9.1 已知技术限制

| 限制 | 影响 | 缓解方案 |
|------|------|---------|
| **WorkerW 为非官方 API** | Windows 大版本更新可能 break | 跟踪 Windows Insider Preview；准备 fallback 为独立窗口模式 |
| **壁纸层无法接收键盘输入** | 壁纸模式下无法打字 | 深度交互（对话）通过独立窗口实现 |
| **壁纸层鼠标交互受限** | 前台有窗口时无法点击壁纸 | 支持独立窗口模式 + 全局热键 |
| **WebView2 内存开销** | 比纯原生渲染占用更多内存 | 控制 DOM 复杂度；PixiJS Canvas 渲染为主，DOM 只用于对话窗口 |
| **tauri-plugin-wallpaper 成熟度** | 可能存在边缘 case | MIT 许可证可 fork 修复；核心代码量不大 |

### 9.2 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 1803+ / Windows 11 |
| 运行时 | WebView2（Tauri 安装包可内嵌） |
| OpenClaw | 已安装并可运行（MVP 不含自动安装） |
| 最低硬件 | 4GB RAM, 集成显卡即可 |

### 9.3 技术选型确认

| 层级 | 技术 | 版本 |
|------|------|------|
| 桌面壳 | Tauri v2 | 2.x |
| 壁纸嵌入 | tauri-plugin-wallpaper | latest |
| 前端框架 | 待定（React / Vue 3 / Solid） | - |
| 2D 渲染 | PixiJS | v8 |
| 角色动画 | Spritesheet 帧动画（PixiJS AnimatedSprite） | - |
| 通信 | OpenClaw Gateway API（HTTP + WebSocket） | - |
| 构建工具 | Vite | latest |
| 包管理 | pnpm | latest |
