## 1. 当前实现状态评估

### 1.1 技术栈实际选型

| 层级 | PRD 设计 | 实际选型 | 偏差说明 |
|------|---------|---------|---------|
| 2D 渲染 | PixiJS v8 | **Phaser 3** | 从纯渲染库改为游戏引擎，获得完整的场景管理、动画系统、Tilemap 支持 |
| 角色动画 | Spritesheet + PaletteSwap | **Spritesheet（Agent Town 素材集）** | 使用 9 套预制角色 spritesheet 代替调色板交换，效果更好 |
| 场景构建 | 手绘 Tilemap | **Tiled 编辑器 tilemap（office2.json）** | 专业 Tilemap 工具，支持 spawns/POIs 对象层 |
| 前端框架 | React | React 19 | 一致 |
| 状态管理 | Zustand | Zustand 5 | 一致 |
| 壁纸嵌入 | tauri-plugin-wallpaper | **暂未集成** | 当前仅独立窗口模式 |

**评价**：技术选型务实，Phaser 3 比 PixiJS 更适合 tilemap + 角色管理场景。Agent Town 素材质量高。

## 2. 优化空间分析

### 2.1 体验优化（高优先级）

| # | 问题 | 描述 | 建议方案 | 预估工时 |
|---|------|------|---------|---------|
| UX-1 | **Session 轮询间隔过短** | 每 3 秒 `refreshSessions()` 全量拉取，不必要的网络开销 | 依赖 WebSocket 事件驱动更新，移除或延长轮询至 30s 作为 fallback | 0.5d |
| UX-2 | **角色消息预览缺失** | 点击角色才能看到信息，缺乏「扫一眼」的信息获取 | 角色头顶显示最近消息的简短气泡（3-5 秒自动消失） | 2d |
| UX-3 | **ChatPanel 无历史加载** | 打开聊天面板只显示实时消息，无法看到历史 | 接入 `sessions.history` API，支持向上滚动加载更多 | 1.5d |
| UX-4 | **离线模式体验单调** | 仅黑色半透明遮罩 + 文字提示 | 角色进入"休眠"姿势（趴桌子/躺椅子），场景灯光变暗，更生动 | 2d |
| UX-5 | **无启动引导** | 首次打开无任何引导，用户不知如何操作 | 简单的首次使用引导：连接配置 → 功能介绍 → 操作提示 | 1.5d |
| UX-6 | **角色点击区域偏小** | 48x48 像素在高 DPI 屏幕上难以准确点击 | 扩大交互 hitArea（sprite 周围增加 padding） | 0.5d |
| UX-7 | **无角色 hover 预览** | 鼠标悬停无反馈，不知道角色是否可交互 | hover 时角色轻微放大 / 高亮 + tooltip 显示名称和状态 | 1d |

### 2.2 性能优化（中优先级）

| # | 问题 | 描述 | 建议方案 | 预估工时 |
|---|------|------|---------|---------|
| PERF-1 | **无前台暂停策略** | 即使被其他窗口完全遮挡，Phaser 仍全速运行 | 实现 `document.visibilitychange` 检测，被遮挡时降帧至 1-2 FPS | 1d |
| PERF-2 | **Phaser scale.RESIZE 频繁触发** | 窗口调整时每帧都执行 `centerCamera()` | 添加 debounce，resize 结束后 200ms 才重算 | 0.5d |
| PERF-3 | **Tileset 资源体积** | 16 张 tileset PNG 全部预加载，但 office2.json 实际只用其中一部分 | 裁剪不使用的 tileset 图片，或合并为 texture atlas | 1d |
| PERF-4 | **角色动画帧率固定** | 所有角色始终播放动画，即使在视口外 | 视口外角色暂停动画更新 | 1d |

### 2.3 稳定性 / Bug 修复（高优先级）

| # | 问题 | 描述 | 建议方案 | 预估工时 |
|---|------|------|---------|---------|
| BUG-1 | **WebSocket 重连后状态不一致** | 重连成功后依赖轮询同步，可能出现短暂角色幽灵/丢失 | 重连成功后立即全量刷新 sessions + agents，强制同步一次 | 0.5d |
| BUG-2 | **座位溢出无降级** | subagent 超过可用座位数时共享最后一个座位，视觉重叠 | 溢出角色排列在休息区/走廊，或使用"站立"姿势 | 1d |
| BUG-3 | **Tray 菜单重建闪烁** | 每次 `update_tray_status` 都 remove + rebuild 整个 tray | 只更新变化的 menu item text，不重建整个 tray | 1d |
| BUG-4 | **InfoBubble 位置跟随** | 角色移动时 InfoBubble 可能不跟随更新 | InfoBubble 绑定到角色 sprite，在 update 循环中同步位置 | 0.5d |

### 2.4 代码质量（低优先级但重要）

| # | 建议 | 预估工时 |
|---|------|---------|
| CODE-1 | 添加 ESLint + Prettier 配置，统一代码风格 | 0.5d |
| CODE-2 | 核心模块单元测试（GatewayClient、SessionMapper、AgentManager） | 3d |
| CODE-3 | E2E 测试框架搭建（WebDriver/Playwright for Tauri） | 2d |
| CODE-4 | 日志系统分级（debug/info/warn/error），线上版去除 console.log | 0.5d |

### 5.4 关键商业洞察

1. **壁纸是 Token 消费的最佳入口**：用户在桌面看到 Agent 工作 → 自然产生对话需求 → 消耗 Token → Token 不足时壁纸场景内弹出充值 → 闭环
2. **情感连接驱动付费**：当用户把 Agent 当作"自己的小助手"，换装/个性化的付费意愿远高于纯工具型产品
3. **壁纸 = 7x24 广告位**：产品在桌面永远可见，是 OpenClaw 生态最好的获客渠道
4. **UGC 是长期壁垒**：参考 Wallpaper Engine 的 Steam Workshop，社区创作的皮肤/场景可以产生飞轮效应

## 6. 竞品差异化分析

### 6.1 竞品对比

| 维度 | Wallpaper Engine | Lively Wallpaper | OpenClaw Wallpaper |
|------|-----------------|------------------|--------------------|
| **定位** | 通用动态壁纸引擎 | 开源免费动态壁纸 | **AI Agent 可视化桌面伴侣** |
| **价格** | $3.99 一次性 | 免费 | 免费（虚拟商品变现） |
| **内容** | 用户上传壁纸 | 用户上传壁纸 | **AI Agent 实时活动** |
| **交互性** | 被动展示为主 | 被动展示为主 | **深度交互：对话、管控、状态同步** |
| **数据源** | 无 | 无 | **实时 Agent 状态** |
| **情感连接** | 低（装饰性） | 低（装饰性） | **高（拟人化 AI 助手）** |
| **许可证** | 商业闭源 | GPL v3 | MIT |
| **技术栈** | C++ / Steam | C# / WinUI 3 | Rust + TypeScript |

### 6.2 核心差异化优势

1. **"活的"壁纸 vs 静态/循环壁纸**  
   竞品壁纸是预设内容循环播放，OpenClaw Wallpaper 的内容是**实时的 AI 活动**。每一天、每一刻场景都不同，因为 Agent 在真正工作。

2. **交互深度不可比**  
   Wallpaper Engine 的交互停留在鼠标跟踪/粒子效果。OpenClaw Wallpaper 支持**与 AI 对话、查看任务状态、管理 AI 服务**——这是功能性交互，不是装饰性交互。

3. **数据驱动的个性化**  
   场景内容由用户的实际 AI 使用数据驱动：用了哪些 Agent、做了什么任务、消耗了多少 Token。这种个性化是竞品无法复制的。

4. **生态绑定**  
   壁纸不是独立产品，是 OpenClaw 生态的**桌面入口**。用户安装壁纸 → 使用 Agent → 购买 Token → 解锁皮肤，形成生态闭环。

5. **技术栈优势**  
   Tauri（Rust）比 Electron 轻 5-10 倍内存，比 C#/WinUI 3 更适合跨平台。MIT 许可证无商业限制。

### 6.3 差异化定位声明

> **OpenClaw Wallpaper 不是一个"更好的壁纸引擎"——它是第一个"AI 团队桌面可视化"产品。**  
> 竞品让你的桌面"好看"，我们让你的桌面"有用"。
